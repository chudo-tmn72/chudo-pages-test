# Комментарии для разработчика — сайт ЧУДО

Это набор ассетов и комментариев по утверждённой дизайн-концепции сайта ЧУДО.

## 1) Общая идея
Сайт должен выглядеть как продолжение оформления VK: ярко, дружелюбно, по-детски, но не инфантильно. Главный вайб — «яркая, стильная детская одежда в наличии; всё понятно, красиво и вызывает доверие».

## 2) Цвета
Основные: 
- розовый / фуксия — главный фон и акцент;
- жёлтый — пушистое «ЧУДО», звёздочки, важные акценты;
- белый — карточки, стикеры, воздух;
- салатовый — точечный дополнительный акцент.

Салатовый использовать дозированно: маленькие сердечки, стрелки, подчёркивания, метки «в наличии», отдельные doodle-элементы, микродетали в иконках. Не делать его главным фоном.

Рекомендуемые опорные цвета (можно слегка уточнить в вёрстке):
- Pink: #FF2D8C
- Yellow: #FFD72E
- White: #FFFFFF
- Lime: #CDEB4A
- Light Pink background: #FFE8F2
- Light Blue background: #DFF0FF
- Light Lilac background: #EFE6FF

## 3) Что обязательно должно быть редактируемым HTML-текстом
Во всех блоках тексты должны быть редактируемыми, НЕ запечёнными в картинки:
- заголовки;
- подзаголовки;
- кнопки;
- подписи карточек;
- описания преимуществ;
- шаги «Как заказать»;
- ростовки и подписи по возрасту.

В графике можно оставлять только декоративные элементы: Лера, мальчик, пушистое «ЧУДО», doodle, мягкие формы, товары/коллажи, иконки, плашки.

## 4) Первый экран (hero)
Использовать:
- `/hero/hero-desktop.webp` — desktop;
- `/hero/hero-mobile.webp` — mobile;
- `/previews/site-mockup-full.webp` — только как общий референс структуры.

Hero = Лера + пушистое жёлтое «ЧУДО» + ярко-розовый фон + мягкая жёлтая подложка + doodle.

Текст на первом экране:
- Заголовок: `Одежда для яркого детства`
- Подзаголовок: `Готовые образы, стильные детали и помощь с размером.`
- Доп. строка: `В наличии в Тюмени.`
- Кнопка: `Перейти в магазин VK`

Текст/кнопка должны быть живыми HTML-элементами.

## 5) Карточки категорий
Основные разделы на сайте сейчас:
1. Девочкам
2. Мальчикам
3. По росту

Подготовленные файлы:
- `/cards/card-girls.webp`
- `/cards/card-boys.webp`
- `/cards/card-growth-reference.webp` (референс)
- `/cards/card-looks-future.webp` (заготовка на будущее, на текущем сайте не показывать)

### Важно по сборке карточек
Так как тексты должны быть редактируемыми, полные карточки выше использовать как визуальный референс.
Для реальной сборки удобно брать правую визуальную часть из:
- `/cards/visual-crops/girls-visual.webp`
- `/cards/visual-crops/boys-visual.webp`
- `/cards/visual-crops/looks-visual.webp`

Левую часть карточек (заголовок, описание, кнопка/стрелка) собрать HTML/CSS.

### Тексты карточек
**Девочкам**
- Заголовок: `Девочкам`
- Описание: `Нежные образы, стильные детали и комфорт на каждый день`

**Мальчикам**
- Заголовок: `Мальчикам`
- Описание: `Удобные модели для активных приключений`

**По росту**
- Заголовок: `По росту`
- Описание: `Легко подобрать размер по росту ребёнка`
- В текущей финальной сборке блок «По росту» делать БЕЗ ребёнка.
- В текущем наборе есть референс-картинка и отдельные ростовые мини-карточки; финальный блок лучше собрать через HTML + мини-карточки + декор.

## 6) Ростовки (зафиксировано)
Использовать именно эти значения:
- `82–92 см` — `примерно на 1,5–2,5 года`
- `92–104 см` — `примерно на 2,5–4 года`
- `104–110 см` — `примерно на 4–5 лет`

Готовые мини-карточки:
- `/cards/growth-82-92.png`
- `/cards/growth-92-104.png`
- `/cards/growth-104-110.png`

Формулировка именно `примерно на ...`, а не жёсткий возраст.

## 7) Блок преимуществ
Иконки отдельно:
- `/icons/icon-measurements.png`
- `/icons/icon-photo-video.png`
- `/icons/icon-size-help.png`
- `/icons/icon-tyumen.png`

Подписи под иконками — HTML-текстом:
1. `Точные замеры` — реальные замеры каждого изделия.
2. `Доп. фото / видео` — по запросу показываем дополнительные материалы.
3. `Помощь с размером` — подскажем лучший размер по росту и особенностям ребёнка.
4. `В наличии в Тюмени` — товары локально, быстрая выдача/доставка.

## 8) Блок «Как заказать»
Иконки отдельно:
- `/icons/icon-step-vk.png`
- `/icons/icon-step-cart.png`
- `/icons/icon-step-delivery.png`

Подписи — HTML-текстом:
1. `Перейдите в магазин VK`
2. `Выберите товар и оформите заказ`
3. `Мы свяжемся и отправим`

## 9) Декор
Использовать ассеты из папки `/decor/` дозированно. Не перегружать каждый блок.

Файлы:
- hearts, stars, crown, bow, arrows, doodle lines, sparkles, flower,
- `blob-white-cloud.png`
- `blob-yellow.webp`
- `sticker-white.png`

Лучший принцип: чистые белые блоки + 1–3 акцентных doodle/стикера на секцию.

## 10) Особые замечания
- Лера — главное лицо бренда.
- Мальчик не должен выглядеть стоковым; ориентироваться на уже выбранный визуал как на «дружелюбного мальчика из того же мира», почти «близкого по вайбу к Лере».
- Раздел `Образы` подготовлен на будущее, но сейчас на странице не выводится.
- Сайт собирается блоками, а не одной большой картинкой.
- Тени мягкие, формы округлые, интерфейс дружелюбный.

## 11) Что внутри архива
- готовые hero-изображения;
- карточки-разделы и визуальные кропы;
- ростовые мини-карточки;
- иконки преимуществ;
- иконки шагов заказа;
- декоративные элементы;
- превью полной концепции и референсные листы.
